# Form And Content Flow

## Use This Reference For

- forms and checkout-like flows
- landing page or onboarding sequence
- progressive disclosure and CTA hierarchy

## Progressive Disclosure

Keep frequent actions visible and advanced options available without overwhelming the user. The goal is not to hide power; it is to reduce the amount of information that competes for attention at once.

Use progressive disclosure when:

- novice and advanced users share the same screen
- a form has optional complexity
- details support the main action but are not needed immediately

## Error Recovery

For important forms, combine:

- field-level error messages
- a summary after failed submission
- preserved user input
- a clear path back to the failed field

## Content Priority

A stable first-screen sequence usually looks like:

1. what this page is
2. why it matters
3. what to do next
4. why the user should trust the action
5. deeper detail and secondary choices

This structure works for landing pages, product pages, and many transactional flows because it lowers cognitive load before asking for commitment.

## Confirmation Design

After success, answer:

- what just happened?
- what changes now?
- what should the user do next?

Avoid generic "done" messages that force the user to reorient manually.

## What To Avoid

- revealing advanced options before the main task is clear
- long forms with no progress cues
- primary and secondary CTAs with equal visual weight
- success pages that behave like dead ends
