# Brandbook Distillation

Turn the raw identity package into a compact working rule set before editing code.

## Extract first

- approved primary mark
- allowed alternate marks
- minimum sizes
- clear-space rules if given
- primary colors and accent colors
- typography families and fallback guidance
- example misuse or prohibited treatments

## Output shape

Write the distilled result as a short repo-safe note with:

- the approved default logo choice
- the exact hex values that matter in code
- the typography direction for web and documents
- any special rules about backgrounds, contrast, or accent use
- anything teams should avoid improvising later

## Important rule

Do not keep reaching back into the raw brand folder during normal implementation work. Distill once, then build from the distilled note and checked-in derivatives.
